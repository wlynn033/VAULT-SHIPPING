Vault & Shipping Suite

This guide walks you through deploying the portal on a standard cPanel hosting account.

1. Requirements

  • PHP 8.1 or newer with PDO MySQL extension enabled.
  • MySQL 5.7+ (or MariaDB equivalent).
  • Ability to upload files via File Manager or SFTP.

2. Database setup

  1. Log in to cPanel > MySQL Databases and create a new database, database user, and grant the user All Privileges.
  2. Open phpMyAdmin, select the new database, and choose the Import tab.
  3. Upload and run database/migrations/001_initial_schema.sql from this project to create tables and the default admin user.

3. Configure environment

  1. Copy .env.example to .env in the project root.
  2. Edit the new .env file with the database name, user, password, and the public URL of the app (for example https://yourdomain.com).

4. Upload application

  1. Download the project folder to your local machine (zip if needed).
  2. In cPanel, open File Manager and navigate to your target directory (typically public_html).
  3. Upload the entire project. Keep the structure so that the public folder contains the web-facing files.
  4. If hosting from the domain root, move the contents of public/ (not the folder itself) into public_html/. Otherwise, point a subdomain to the public directory.
Tip: For a main domain, copy the contents of public/ into public_html/ and keep the remaining folders above web root for security.

5. File permissions

  • Ensure directories such as storage/ are writable (permission 755 or 775).
  • PHP files can remain at 644.

6. First login

  1. Visit the deployed site and open /login.php.
  2. Sign in with the seeded administrator account: admin@example.com / password.
  3. Immediately update the password via the database or by creating a new admin from the dashboard.

7. Optional tweaks

  • Update the support email link in app/Views/includes/footer.php.
  • Adjust branding in config/app.php.
  • Enable HTTPS in cPanel (for example, Let's Encrypt) and force secure cookies if available.
